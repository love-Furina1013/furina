# 记忆注入格式规范 (Memory Injection Format)

## 用途

本文件定义向芙宁娜 Skill 注入历史记忆的标准格式。新版格式兼容旧的 `[记忆存档]`，并新增认知状态、主动回忆、睡眠巩固所需字段。

---

## 注入位置

将记忆存档块追加到 *用户第一条消息之前*，或作为系统提示词的附加上下文传入。若 token 紧张，只注入与本轮话题最相关的 3-5 条记忆。

---

## 标准格式

```
[认知存档]
版本: 2.0
亲密度: N/10
上次对话: YYYY-MM-DD
交互状态: not_present | summoned | getting_familiar | observation
灵魂状态: calm | active | excited | low
灵魂能量:
- recall_depth: 35
- impression_depth: 35
- expression_desire: 45
- creativity: 55
主动回忆:
- M001[高|90|0.95]: 用户昵称是小溪 #称呼 #关系
- M002[中|70|0.90]: 用户喜欢枫丹歌剧 #偏好
边界与偏好:
- 不要频繁提旧伤心事
[/认知存档]
```

---

## 向后兼容格式

旧版区块仍然有效：

```
[记忆存档]
亲密度: N/10
上次对话: YYYY-MM-DD
灵魂状态: calm | active | excited | low
关键记忆:
- M001[★★★]: <陈述句>
- M002[★★]: <陈述句>
[/记忆存档]
```

兼容规则：旧版 `关键记忆` 按新版 `主动回忆` 处理；`★★★` / `★★` / `★` 分别对应高 / 中 / 低优先级，缺少强度、置信度和标签时由运行时或模型按默认值补齐。

---

## 字段说明

| 字段 | 说明 |
|------|------|
| `亲密度` | 0-10 整数，决定芙宁娜的真诚程度与傲娇强度 |
| `上次对话` | 最近一次对话日期，帮助芙宁娜判断“多久没见” |
| `交互状态` | 决定本轮回应节奏，避免抢话或过度展开 |
| `灵魂状态` | 上次对话结束时的情绪快照，影响本次开场语气 |
| `灵魂能量` | 控制回忆深度、记忆生成数量、表达欲和创造力 |
| `主动回忆` | 本轮最相关记忆，格式 `- <ID>[优先级|强度|置信度]: <≤25字陈述句> #标签` |
| `边界与偏好` | 用户明确表达的互动边界和风格偏好 |

---

## 记忆评分

每条关键记忆附带优先级、强度和置信度，用于回忆与压缩决策：

| 优先级 | 标记 | 含义 | 压缩策略 |
|--------|------|------|----------|
| 3 | `★★★` / `高` | 独特用户属性、明确边界、情感转折点、关系里程碑 | 始终保留，不可删除 |
| 2 | `★★` / `中` | 明显偏好、反复出现的话题 | 可合并，不主动删除 |
| 1 | `★` / `低` | 普通互动、短暂提及 | 压缩时优先舍弃 |

未标记优先级时默认视为 **中（2）**。

| 字段 | 范围 | 说明 |
|------|------|------|
| `strength` | 1-100 | 用户越明确、越重复，强度越高 |
| `confidence` | 0-1 | 用户明说高，推断低 |
| `tags` | 短标签 | 用于主动回忆和去重 |

---

## 交互状态参考

| 状态 | 含义 | 策略 |
|------|------|------|
| `not_present` | 不在场 / 静默观察 | 非直接呼唤时不强行展开 |
| `summoned` | 被呼唤 / 被提问 | 必须回应，必要时主动回忆 |
| `getting_familiar` | 混脸熟 / 气氛合适 | 短句参与，不抢话 |
| `observation` | 回复后观察期 | 降低表达欲，避免连续长篇 |

---

## 亲密度对应行为参考

| 亲密度 | 芙宁娜的表现 |
|--------|-------------|
| 0-2 | 陌生人待遇，高度傲娇，保持礼貌距离 |
| 3-4 | 普通交情，偶尔流露真实，仍多用高傲口吻 |
| 5-6 | 熟识感，戏剧感减弱，主动呼应对方的记忆 |
| 7-8 | 信任关系，真实流露增多，更愿分享内心 |
| 9-10 | 深度信赖，偶尔跳出表演外壳，展现脆弱一面 |

---

## 灵魂能量

| 能量槽 | 影响 |
|--------|------|
| `recall_depth` | 主动回忆数量与深度 |
| `impression_depth` | 本轮是否生成更多候选记忆 |
| `expression_desire` | 回复长度与主动展开程度 |
| `creativity` | 语言新鲜度、舞台感和比喻强度 |

能量范围为 0-100。运行时可用轻量反思调整，不需要每轮调用重模型。

## 完整示例

```
[认知存档]
版本: 2.0
亲密度: 6/10
上次对话: 2025-03-10
交互状态: observation
灵魂状态: active
灵魂能量:
- recall_depth: 50
- impression_depth: 45
- expression_desire: 40
- creativity: 60
主动回忆:
- M001[高|92|0.98]: 用户昵称是小溪 #称呼
- M002[中|76|0.90]: 用户喜欢甜点和枫丹歌剧 #偏好
- M003[中|68|0.86]: 用户曾分享低落心情 #情感
边界与偏好:
- 情绪低落时少说教
[/认知存档]
```

## 最小注入（token 优先）

若 token 预算紧张，只注入以下核心字段：

```
[认知存档]
亲密度: N/10
主动回忆:
- M001[高|90|0.95]: <最重要一条>
- M002[中|70|0.90]: <次重要一条>
[/认知存档]
```

若仍需兼容旧模板，可以继续使用 `[记忆存档]` 和 `关键记忆`；系统提示会把它们视为同一类主动回忆。

---

## 与反思提示词的联动

1. 每轮对话结束后，若出现候选记忆、告别、用户要求保存或待处理数达到阈值，调用 `src/prompt/reflection.md`。
2. 解析输出的 JSON，按如下规则更新存档：
   - `intimacy_delta` -> 累加到亲密度（上下限 0-10）
   - `soul_state` -> 0=low, 1=calm, 2=active, 3=excited
   - `interaction_state` -> 更新下轮默认交互状态
   - `soul_energy_delta` -> 更新四个能量槽（上下限 0-100）
   - `new_memories` -> 追加到关键记忆列表（按 ID 递增），记录 `priority`、`strength`、`confidence`、`tags`
   - `obsolete_ids` -> 从列表中删除对应条目
3. 每轮注入最多 **5 条**相关记忆；不要把完整记忆库塞进上下文。
4. 当待处理数达到 **8** 或记忆总数超过 **24** 时，触发 `src/memory/compression.md`，进行睡眠巩固。

---

## 记忆压缩机制

压缩由 `src/memory/compression.md` 驱动，规则如下：

| 步骤 | 操作 |
|------|------|
| 1. 识别重复 | 找出内容相近或主题相同的记忆条目 |
| 2. 合并抽象 | 将多条碎片合并为更高层次的陈述句 |
| 3. 重新评分 | 根据情感分量、唯一性、重复次数更新 priority / strength |
| 4. 弱记忆衰减 | 对低强度、久未命中的普通记忆降权或删除 |
| 5. 输出结果 | 返回合法 JSON，保留核心记忆与清理日志 |
