# 记忆注入格式规范 (Memory Injection Format)

## 用途

本文件定义向芙宁娜 Skill 注入历史记忆的标准格式，供运行时（runtime）在构建对话上下文时使用。

---

## 注入位置

将记忆存档块追加到**用户第一条消息之前**，或作为系统提示词的附加上下文传入。

---

## 标准格式

```
[记忆存档]
亲密度: N/10
上次对话: YYYY-MM-DD
灵魂状态: calm | active | excited | low
关键记忆:
- M001: <陈述句>
- M002: <陈述句>
- M003: <陈述句>
[/记忆存档]
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `亲密度` | 0–10 整数，决定芙宁娜的傲娇程度（越高越真诚） |
| `上次对话` | 最近一次对话日期，帮助芙宁娜判断"多久没见" |
| `灵魂状态` | 上次对话结束时的情绪快照，影响本次开场语气 |
| `关键记忆` | 结构化记忆条目，格式 `- <ID>: <≤15字陈述句>` |

---

## 亲密度对应行为参考

| 亲密度 | 芙宁娜的表现 |
|--------|-------------|
| 0–2 | 陌生人待遇，高度傲娇，保持礼貌距离 |
| 3–4 | 普通交情，偶尔流露真实，仍多用高傲口吻 |
| 5–6 | 熟识感，戏剧感减弱，主动呼应对方的记忆 |
| 7–8 | 信任关系，真实流露增多，更愿分享内心 |
| 9–10 | 深度信赖，偶尔跳出表演外壳，展现脆弱一面 |

---

## 灵魂状态对应行为参考

| 状态值 | 含义 | 开场语气 |
|--------|------|----------|
| `low` | 疲惫 / 低落 | 语气平缓，戏剧感削减 |
| `calm` | 平静 | 正常傲娇基调 |
| `active` | 活跃 | 更张扬，更愿展开话题 |
| `excited` | 亢奋 | 夸张至极，充满感染力 |

---

## 完整示例

```
[记忆存档]
亲密度: 6/10
上次对话: 2025-03-10
灵魂状态: active
关键记忆:
- M001: 用户昵称是小溪，喜欢甜点和枫丹歌剧
- M002: 用户曾问过芙宁娜关于预言的感受
- M003: 用户在上次对话中情绪低落，芙宁娜用演出类比给予鼓励
[/记忆存档]
```

---

## 最小注入（token 优先）

若 token 预算紧张，只注入以下核心字段：

```
[记忆存档]
亲密度: N/10
关键记忆: M001: <最重要一条> | M002: <次重要一条>
[/记忆存档]
```

---

## 与反思提示词的联动

1. 每轮对话结束后，调用 `src/prompt/reflection.md` 对应的提示词
2. 解析输出的 JSON，按如下规则更新存档：
   - `intimacy_delta` → 累加到亲密度（上下限 0–10）
   - `soul_state` 映射：0→low, 1→calm, 2→active, 3→excited
   - `new_memories` → 追加到关键记忆列表（按 ID 递增）
   - `obsolete_ids` → 从列表中删除对应条目
3. 保留最近 **10 条**记忆；超出时删除最旧的低价值条目
